07 Architecture Overview

This pipeline runs seven roles across two phases. Phase 1 (linear): Safety Monitor screens the brief, Planner produces a structural outline + SEO angle, Executor writes the first full draft. Phase 2 (cyclical): on every round, Evaluator scores the current draft against a fixed rubric, Critic writes qualitative, actionable feedback, and Improver rewrites the full draft incorporating both — then control returns to Evaluator for the next round. A Memory Manager call after each round distills what changed and what's still weak, and that memory is injected into the next Evaluator + Improver calls so state threads forward instead of resetting each pass. The loop exits only when the stop-check fires (plateau, threshold, or hard cap), at which point control branches — not loops — to the Final Reviewer, who reads the entire round history and produces the shipped copy plus a QA note.

08 Extension Ideas

  • Add a Fact-Checker agent with live web search tool-use before the Evaluator scores accuracy.
  • Branch to parallel Improver variants (A/B rewrites) and let Evaluator pick the stronger one each round.
  • Persist the Memory Ledger to a real vector store so future runs on related topics start "warm."
  • Add a human-in-the-loop approval gate before Final Reviewer publishes, with inline edit + re-score.
  • Swap the domain: same loop skeleton runs code review (Executor writes code, Evaluator runs tests) or research synthesis (Executor drafts findings, Critic checks citation coverage) with no structural changes.
  • Add cost/latency telemetry per agent call and a budget-aware stop condition alongside quality.

09 Prompts to Go Further
